# RemoteSync

RemoteSync 是一个轻量级、高效的远程文件同步工具，旨在简化本地开发环境与远程服务器之间的数据同步流程。它通过 SSH 安全连接，提供增量对比、并行上传及灵活的配置管理。

## 🚀 主要特性

- **交互式 CLI**: 拥有美观且直观的终端交互界面。
- **智能对比**: 自动分析本地与远程文件差异（基于文件大小与修改时间），仅同步变更部分。
- **并行同步**: 支持多任务并行上传，极大提升大批量小文件的同步效率。
- **多环境支持**: 支持多服务器、多路径组合（Groups）配置，轻松管理复杂的部署需求。
- **差异清理**: 提供远程冗余文件清理功能，确保目标目录与本地保持一致。
- **多种认证**: 支持 SSH 密码及私钥文件认证方式。

## 🛠️ 安装

### 从源码编译

确保你的系统中已安装 Go 1.20 或更高版本。

```bash
git clone <repository-url>
cd remotesync
go build -o remotesync.exe
```

## 📖 使用指南

1. **运行程序**:
   执行编译生成的二进制文件：
   ```bash
   ./remotesync.exe
   ```

2. **初始化配置**:
   首次运行将引导你添加服务器信息。配置信息默认保存在用户目录下的 `~/.remotesync/config.yaml`。

3. **同步流程**:
   - 选择目标服务器。
   - 选择预定义的路径组合（Group）。
   - 查看差异对比报告。
   - 确认执行同步更新。

## 📁 项目结构

- `main.go`: 程序入口，负责初始化与启动。
- `config/`: 负责配置文件的加载、保存与管理。
- `ssh/`: 核心 SSH/SFTP 客户端封装。
- `ui/`: 交互式命令行界面的实现。
- `compare/`: 核心差异算法，实现文件状态对比。
- `upload/`: 负责具体的文件上传与任务调度。

## 📦 发布与部署

本项目使用 GitHub Actions 实现了自动化流水线：

- **持续集成**: 每当推送至 `master` 分支时，会自动在 Windows、Linux 和 macOS 平台上进行构建验证。
- **自动发布**: 当你推送以 `v` 开头的标签（例如 `v1.0.0`）时，工作流会自动创建 GitHub Release，并将所有平台的二进制文件作为附件上传。

**如何触发发布**:
```bash
git tag v1.0.0
git push origin v1.0.0
```
